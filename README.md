# Система управления библиотекой (Library Management System)

---
### Назначение
Система управления библиотекой (Library Management System, LMS) — это комплексное программное решение для автоматизации основных процессов современной библиотеки. Система предназначена для замены бумажного документооборота, повышения эффективности работы сотрудников и улучшения качества обслуживания читателей.

---

### Основной функционал
- **Управление библиотечным фондом**
  - Ведение электронного каталога книг
  - Учет экземпляров (инвентарные номера, статусы)
  - Организация каталога по разделам: автор, название, жанр, ISBN
  - Учет издательств и годов издания

- **Управление пользователями**
  - Ведение базы данных читателей
  - Учет читательских билетов и учетных записей
  - История посещений и операций

- **Операции выдачи и возврата**
  - Оформление выдачи книг с фиксацией дат
  - Прием возвращенных книг
  - Автоматический расчет штрафов за просрочку
  - Контроль сроков возврата

- **Поиск и аналитика**
  - Мощный поисковый интерфейс
  - Формирование отчетов и статистики
  - Аналитика популярности книг и авторов

### Взаимодействие с внешними системами

| Внешняя система | Тип взаимодействия | Назначение |
|----------------|-------------------|------------|
| **Платежный шлюз** | API интеграция | Автоматическое списание штрафов, онлайн-оплата |
| **Email-сервис** | API интеграция | Отправка уведомлений и напоминаний |
| **SMS-шлюз** | API интеграция | СМС-уведомления о статусах и штрафах |
| **Национальный каталог** | API/импорт данных | Загрузка библиографических данных по ISBN |

---

## Акторы системы
*   **Библиотекарь (Librarian)** — основной пользователь системы. Выполняет все операции по управлению фондом и выдачей книг.
*   **Читатель (Reader)** — пользуется системой для поиска книг и просмотра своей учетной записи.
*   **Каталог (Catalog)** — внешняя система, с которой взаимодействует наша система. Пример **внешнего сервера** или **базы данных**. Является актором, потому что система активно с ним взаимодействует.

## Список прецедентов и их описание

| Прецедент (Use Case) | Актор | Описание |
| :--- | :--- | :--- |
| **Аутентифицироваться (Authenticate)** | Библиотекарь, Читатель | Вход пользователя в систему под своим логином и паролем. |
| **Управлять учетными записями читателей** | Библиотекарь | Добавление, удаление, редактирование учетных записей читателей. |
| **Добавить книгу в каталог** | Библиотекарь | Внесение информации о новой книге в библиотечный фонд. |
| **Удалить книгу из каталога** | Библиотекарь | Списание книги из фонда. |
| **Оформить выдачу книги** | Библиотекарь | Фиксация факта выдачи книги читателю. Включает в себя прецедент **"Проверить наличие книги"**. |
| **Принять возврат книги** | Библиотекарь | Фиксация факта возврата книги от читателя. |
| **Проверить наличие книги** | Библиотекарь, Читатель | Поиск книги по названию, автору или ISBN в каталоге. Является частью (`include`) прецедентов **"Оформить выдачу книги"** и **"Найти книгу"**. |
| **Найти книгу** | Читатель | Поиск и просмотр информации о доступных книгах. |
| **Посмотреть свою учетную запись** | Читатель | Просмотр читателем информации о себе: текущие взятые книги, история выдач, сроки возврата. |
| **Обновить каталог** | Каталог | Внешнее взаимодействие. Система отправляет запрос каталогу на обновление информации о книгах (например, после выдачи или возврата). |

---

## Примечания
*   **Включение (`Include`)**:
    *   Прецедент **"Оформить выдачу книги"** ВКЛЮЧАЕТ в себя прецедент **"Проверить наличие книги"**. Нельзя оформить выдачу, не проверив наличие.
    *   Прецедент **"Найти книгу"** ВКЛЮЧАЕТ в себя прецедент **"Проверить наличие книги"**. Поиск по каталогу всегда подразумевает проверку наличия.

*   **Обобщение (Generalization)**:
    *   Акторы **Библиотекарь** и **Читатель** являются обобщением актора **Пользователь (User)**, так как оба они входят в систему. Это отношение показывает, что они наследуют базовые свойства пользователя (например, необходимость аутентификации).
